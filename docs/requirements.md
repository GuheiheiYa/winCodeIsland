# 需求文档

## 1. 项目背景

**Vibe Notch** 是一个受 macOS Dynamic Island（灵动岛）启发的 Windows 桌面应用。开发者在同时使用多个 AI 助手（Claude Code、OpenAI Codex、Google Gemini 等）进行终端开发时，需要一种不打扰工作流的方式来实时监控各会话的状态——谁在运行、谁在等待输入、谁已经完成。

传统方案是频繁切换终端标签页，效率低下且容易打断心流。Vibe Notch 以一条悬浮在屏幕边缘的 Pill 形小条提供"一瞥即知"的状态概览，点击后展开为完整面板展示所有会话详情。

## 2. 目标用户

- 同时使用多个 AI 编程助手的开发者
- 习惯终端工作流（Ghostty、iTerm2 等）的工程师
- 需要监控长时间运行任务状态的用户
- Windows 平台用户（未来可能扩展至 macOS/Linux）

## 3. 功能需求

### FR-1 灵动岛主窗口
- **FR-1.1** 应用启动后显示一条无边框、置顶的 Pill 形小条悬浮于屏幕顶部中央
- **FR-1.2** 窗口尺寸：收起状态 300x36px，展开状态 560x370px（固定高度，内容滚动）
- **FR-1.3** 窗口支持鼠标拖拽移动位置
- **FR-1.4** 拖拽结束时自动吸附到最近的屏幕边缘（上/下/左/右），吸附阈值 40px
- **FR-1.5** 窗口在所有工作区可见，不占用任务栏

### FR-2 收起状态（Collapsed Bar）
- **FR-2.1** 左侧用 Canvas 2D 绘制像素风小章鱼吉祥物，根据会话状态播放不同动画
  - 空闲：睡觉呼吸 + 浮动 "Zzz"
  - 运行中：弹跳 + 手臂打字 + 键盘闪烁
  - 告警：跳跃 + 手臂挥舞 + 红色光晕 + 感叹号
- **FR-2.2** 状态文字实时联动（运行中绿色 / 请确认红色 / 休息中白色）
- **FR-2.3** 文字带科技像素风样式（等宽字体 + 发光效果 + 动态省略号）
- **FR-2.4** 右侧显示总会话数量
- **FR-2.5** 点击后展开为完整面板

### FR-3 展开面板（Expanded Panel）
- **FR-3.1** 顶部栏包含：Canvas 章鱼吉祥物、ALL / STA / CLI 标签切换、音量按钮、关闭按钮
- **FR-3.2** 按 AI 助手类型分组显示会话（Claude / Codex / Gemini）
- **FR-3.3** 每个会话卡片显示：像素风项目图标、项目名称、会话编号、终端类型、状态指示、最近终端输出
- **FR-3.4** 终端输出支持语法高亮（命令灰色、输出绿色、链接蓝色、思考中光标闪烁）
- **FR-3.5** 空状态时显示友好提示
- **FR-3.6** 列表项带交错进入动画
- **FR-3.7** 点击外部区域或关闭按钮收起面板

### FR-4 会话状态管理
- **FR-4.1** 支持六种会话状态：thinking（思考中）、tool_use（执行工具）、responding（生成回复）、working（处理中）、waitingApproval（等待确认）、sleeping（休眠中）
- **FR-4.2** 状态变更时实时同步到 UI
- **FR-4.3** 支持按状态过滤（ALL / STA / CLI）

### FR-5 系统托盘
- **FR-5.1** 任务栏托盘图标支持右键菜单
- **FR-5.2** 菜单项：显示/隐藏窗口、开机启动、退出应用

### FR-6 设置面板（暂不启用）
- **FR-6.1** 预留配置项：开机自启、贴边吸附、主题、快捷键、透明度
- **FR-6.2** 设置面板组件保留但当前版本不再渲染（未来版本恢复）

### FR-7 终端窗口聚焦
- **FR-7.1** 点击展开面板中的会话卡片，自动聚焦到对应的终端窗口
- **FR-7.2** 支持 Windows Terminal（通过 `wt focus-tab` 切换标签页）和传统 CMD/PowerShell（通过 Win32 API 激活窗口）
- **FR-7.3** 通过会话的 `pid` 字段定位并激活终端进程

### FR-8 数据接入（未来）
- **FR-8.1** 支持从真实终端会话抓取数据（node-pty 集成）
- **FR-8.2** 支持多显示器环境下正确识别工作区

## 4. 非功能需求

### NFR-1 性能
- 渲染进程动画帧率稳定 60fps
- 主进程 IPC 通信延迟 < 16ms
- 内存占用控制在 150MB 以内

### NFR-2 视觉
- 全局毛玻璃效果（backdrop-filter blur）
- 暗黑主题为主，预留浅色主题扩展
- 所有动画使用 CSS transitions / Canvas rAF，禁用 setState 式频繁重渲染

### NFR-3 兼容性
- Windows 10/11 支持
- Electron 28+ 稳定运行
- Node.js 18+ 构建环境

### NFR-4 可维护性
- TypeScript 严格类型检查
- 组件职责单一，逻辑通过 Pinia store 集中管理
- Canvas 渲染引擎与业务逻辑解耦
