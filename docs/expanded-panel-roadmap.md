# 展开面板功能拆解文档

本文档针对 **ExpandedPanel（展开状态面板）** 及其子组件进行功能拆解，涵盖已实现功能、待开发项和交互设计。

---

## 1. 架构总览

```
ExpandedPanel (展开面板容器)
├── TopBar (顶部控制栏)
│   ├── 标签过滤 (ALL / STA / CLI)
│   ├── 音量按钮 (预留)
│   ├── 设置按钮 → 打开 SettingsPanel
│   └── 关闭按钮 → 收起面板
├── AgentGroup - Claude (当前仅 Claude 分组)
│   ├── 助手标题 (claude.png 图标 + 名称 + 数量)
│   └── SessionCard × N (会话卡片)
│       ├── 项目图标 (像素风章鱼 SVG，与 Canvas 吉祥物一致)
│       │   ├── sleeping: 闭眼睡觉 + Zzz
│       │   ├── working: 睁眼打字 + 键盘
│       │   └── thinking: 睁大眼睛 + 感叹号
│       ├── 项目信息 (名称 + 编号)
│       ├── 元信息 (时间 + 终端类型)
│       └── 终端输出 (TerminalOutput)
└── EmptyState (空状态提示)

SettingsPanel (设置模态框，独立 overlay)
├── 开机启动 Toggle
├── 贴边吸附 Toggle
├── 主题选择 (Dark / Light / Auto)
├── 快捷键显示
└── 版本信息
```

> **注**：当前版本仅保留 Claude 分组，Codex / Gemini 分组静态数据已移除。分组图标使用 `public/resources/cli-icons/claude.png`。

## 4. 数据流与状态管理

### 4.1 当前数据流

```
Main Process (模拟数据)
    │ IPC: sessions:update
    ▼
Renderer: Pinia Store (notchStore.ts)
    │ reactive
    ▼
Vue Components
    - ExpandedPanel → AgentGroup → SessionCard
    - TopBar (activeTab 过滤)
    - SettingsPanel (settings)
```

### 4.2 目标数据流 (v1.1+)

```
Terminal (node-pty)
    │ 输出流
    ▼
Main Process: Session Parser
    - 识别 AI 助手类型
    - 提取输出内容
    - 状态推断 (working/thinking/sleeping)
    │ IPC: sessions:update / session:add / session:remove
    ▼
Renderer: Pinia Store
    │
    ├──→ ExpandedPanel (会话列表)
    ├──→ CollapsedBar (状态总览)
    └──→ SettingsPanel (持久化配置)
```

| TopBar | — | `collapse` |
